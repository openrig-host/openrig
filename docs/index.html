Purpose Built

A Digital Hardware Engine for Keyboardists

No virtual cable spaghetti, no node graphs, no unpredictable crashes mid-solo.

01

Linear Rack Architecture

Vertical channel strips with direct physical faders, mutes, routing, and bypass controls you already understand.

02

Dual Independent Mixes

Separate FOH (Front of House) and IEM (In-Ear Monitor) output buses per slot so your stage monitoring never compromises house sound.

03

Atomic Song Preloading

Next song patches load off-thread in the background and swap atomically. If a patch load fails, your current rig continues seamlessly.

04

Arm & Wiggle MIDI Learn

Bind CC knobs or restrict key zones in seconds by simply playing a note or turning a control on your keyboard.

05

Next-Block Panic Protection

Instant hardware-fail-safe panic button flushes all active MIDI notes inside the very next audio block.

06

SEH Plugin Exception Shielding

Misbehaving VST3 plugins are caught and isolated per slot, keeping the engine and audio thread alive under all conditions.

⚠️ Stage Preparation Recommendation

OpenRig is built for live performance stability. For seamless shows, follow these stage preparation habits:

  • Rehearse full setlists using your stage laptop, target VST3 plugins, and MIDI controllers.
  • Validate rapid song transitions, scene triggers, and panic buttons during rehearsal.
  • Keep a backup plan (e.g. secondary system or hardware synth fallback) for mission-critical gigs.
Comprehensive Toolkit

Engineered for Real-World Workflows

Every feature focused on speed, clarity, and precision on stage.

Songs, Scenes & Setlists

Complete patch snapshots stored as clean, versioned JSON. Rapidly jump between song sections (Verse / Chorus / Bridge) via MIDI triggers.

Built-In Channel Strip DSP

Integrated noise gate, parametric EQ, compressor, chorus, and reverb available per slot without adding external plugins.

Sample & Waveform Editor

Internal sample player with sample-accurate handles, trim editing, root key mapping, and octave harmonizing.

Arpeggiator & MIDI FX

Up/Down/Random arpeggiators with 0–4 octave range, gate time control, transposer, and BPM sync.

Atomic File System

Saves write to temp files first with auto-rotated backups (`.bak`). Power loss or crashes will never corrupt your setlists.

Absolute Data Sovereignty

No cloud sync, no accounts, no telemetry. Your patches are standard human-readable JSON files you fully control.

1
Background Build
SetlistManager triggers `RigBuilder` on a separate background worker thread.
2
Silent Validation
New VST3 instances are instantiated and validated with a silent `processBlock`.
3
Atomic Pointer Swap
Pointers are swapped under callback lock in < 1ms without audio allocation.
4
Rollback Safety
If a background build fails, the swap never happens and the current rig stays live.
